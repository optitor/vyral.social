# Internationalization (i18n)

Fumadocs supports internationalization through Next.js's built-in i18n features.

## Configure i18n

Next.js provides built-in support for internationalized routing. Configure it in your `next.config.js`:

```js title="next.config.js"
module.exports = {
  i18n: {
    locales: ['en', 'fr', 'es'],
    defaultLocale: 'en',
  },
}
```

## Organizing Translated Content

Create separate directories for each locale:

```
content/
├── docs/           # English (default)
│   └── index.mdx
├── docs.fr/        # French
│   └── index.mdx
└── docs.es/        # Spanish
    └── index.mdx
```

## Translation Workflow

1. **Create locale directories** - Duplicate your content structure for each language
2. **Translate content** - Update MDX files with translated text
3. **Update navigation** - Ensure `meta.json` files are translated
4. **Test locally** - Use locale parameter to test translations

## Accessing Localized Routes

Your localized site will be accessible at:
- English: `http://localhost:3000/docs`
- French: `http://localhost:3000/fr/docs`
- Spanish: `http://localhost:3000/es/docs`

## Build your localized site

Build your site for a specific locale:

```bash
npm run build -- --locale fr
```

Or build your site to include all the locales at once:

```bash
npm run build
```
